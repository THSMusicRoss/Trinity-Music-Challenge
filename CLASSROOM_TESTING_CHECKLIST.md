# Classroom Testing Checklist

## Before class

- [ ] Open the app in the intended classroom browser.
- [ ] Confirm the 500-question bank loads.
- [ ] Test Presentation Mode with the projector.
- [ ] Confirm text is legible from the back of the room.
- [ ] Confirm browser audio is unmuted.
- [ ] Test at least one sound cue.
- [ ] Create the expected number of teams.
- [ ] Confirm keyboard number keys trigger the correct buzzers.
- [ ] Confirm the selected question filters provide enough questions.

## Manual Mode test

- [ ] Award a correct answer to each team.
- [ ] Mark a question unanswered.
- [ ] Reveal an answer.
- [ ] Skip a question.
- [ ] Pause and resume.
- [ ] Adjust a team's score.

## Buzzer Mode test

- [ ] Confirm the first team locks the others.
- [ ] Mark a buzzed team incorrect.
- [ ] Confirm steals reopen when enabled.
- [ ] Reset buzzers.
- [ ] Test number-key buzzers.
- [ ] Confirm an incorrect-answer penalty is applied when enabled.

## Persistence test

- [ ] Begin a game and answer several questions.
- [ ] Refresh the browser.
- [ ] Resume the game.
- [ ] Confirm scores, question position, timer, and revealed answer are restored.
- [ ] Finish the game and confirm Resume Game is cleared.

## Tie test

- [ ] Test Shared Victory.
- [ ] Test Teacher Decides.
- [ ] Test Sudden Death.

## Accessibility test

- [ ] Navigate major controls with Tab and Enter.
- [ ] Confirm visible keyboard focus.
- [ ] Test browser reduced-motion mode.
- [ ] Test increased contrast if supported.
- [ ] Confirm important events are announced with a screen reader.

## Release decision

- [ ] No blocking errors found.
- [ ] No score calculation errors found.
- [ ] No duplicate question appears within one game.
- [ ] Teacher can recover from common mistakes.
- [ ] Projector layout remains readable during full gameplay.
